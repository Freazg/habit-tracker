## Опис проєкту

Habit Tracker - веб-застосунок для відстеження та формування корисних звичок. Система допомагає користувачам створювати, відстежувати та аналізувати свої щоденні звички з детальною статистикою та мотиваційними елементами.

## Основна ідея

Формування звичок - складний процес, який потребує постійного моніторингу та мотивації. Більшість людей намагаються впровадити нові звички, але не мають інструментів для систематичного відстеження прогресу. Застосунок вирішує цю проблему через візуалізацію досягнень, систему streak (серій виконань без пропусків) та детальну аналітику поведінкових патернів.

## Цільова аудиторія

Застосунок призначений для людей, які хочуть сформувати нові корисні звички або позбутися шкідливих. Це можуть бути студенти, що прагнуть покращити навчальну дисципліну, спортсмени, які відстежують тренування, або просто люди, що люблять систематизувати своє життя через метрики та статистику.

## Основний функціонал

Мінімально життєздатний продукт включає управління звичками (створення, редагування, видалення), систему категорій (здоров'я, навчання, спорт, фінанси), щоденне відстеження виконання через календарний інтерфейс та базову статистику (кількість виконань, поточний streak, відсоток успішності).

Розширений функціонал передбачає систему досягнень та бейджів за milestone, детальну аналітику з графіками прогресу, можливість встановлення цілей, систему нагадувань та соціальні функції для обміну досягненнями.

## Технічний стек

Backend буде реалізовано на Node.js з використанням Express.js фреймворку. Для зберігання даних обрано PostgreSQL з Prisma ORM. Валідація даних виконується через Zod, аутентифікація через JWT токени з bcrypt для хешування паролів.

Frontend планується на React.js з Tailwind CSS для стилізації та Zustand для state management (можливо буде змінено після Lab 1).

Для забезпечення якості коду використовуються ESLint та Prettier, для автоматизації - Husky git hooks. Тестування покривається Jest та Supertest. Контейнеризація через Docker, CI/CD через GitHub Actions, деплой на Railway або Render.

## Структура даних

Основні сутності бази даних включають User (користувач з полями id, email, password, name, timezone), Habit (звичка з прив'язкою до користувача, містить назву, опис, категорію, частоту), HabitLog (записи виконання звичок з датою та статусом) та Category (категорії звичок з назвою та кольором).

Зв'язки між сутностями: один користувач має багато звичок, одна звичка має багато записів виконання, одна категорія може бути у багатьох звичок.

## Користувацькі сценарії

Типовий сценарій використання починається з реєстрації або входу користувача в систему. Після входу користувач створює нову звичку, вказуючи назву (наприклад "Ранкова пробіжка"), категорію (Спорт) та частоту (щоденно). Система зберігає звичку та відображає її в списку активних.

Щоденний сценарій полягає у тому, що користувач заходить в застосунок, бачить список звичок на поточний день та відмічає виконані. При відмітці система автоматично оновлює streak (якщо звичка виконувалась вчора, streak збільшується, якщо була перерва - обнуляється) та загальну статистику.

Аналітичний сценарій дозволяє користувачу переглянути прогрес за обраний período. Система візуалізує дані через календар (heat map показує дні виконання), графіки трендів та числові метрики (відсоток виконання, середня кількість виконань на тиждень).

## Критерії вибору технологій

Node.js обрано як runtime через хорошу підтримку асинхронних операцій, що критично для веб-застосунків з багатьма одночасними користувачами. Express.js - мінімалістичний фреймворк, що дозволяє швидко створювати REST API без зайвих абстракцій.

PostgreSQL обрано як реляційну базу даних через потребу в строгій структурі даних, ACID-транзакціях та відмінній підтримці роботи з датами та часовими зонами. Prisma ORM забезпечує type-safety, автоматичну генерацію TypeScript типів та зручні міграції.

GitHub Actions використовується для CI/CD через безкоштовність для публічних репозиторіїв та нативну інтеграцію з GitHub. Docker забезпечує консистентність середовища розробки та продакшену.
